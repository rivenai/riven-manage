MCP Tool Calling

Execute MCP tools directly or let AI models call them automatically.

Riven AI supports both direct tool invocation and AI-driven tool calling where language models decide which tools to use based on user requests.

Overview

MCP (Model Context Protocol) tools extend AI capabilities by allowing models to interact with external services like GitHub, Slack, databases, and more. Riven AI acts as a gateway that routes tool calls to the appropriate MCP servers.

Basic Tool Call Flow

The typical flow for AI-driven tool calling:

1

User sends a request

"Create a GitHub issue for the login bug"

2

AI model decides to use a tool

Model returns tool_calls with github.create_issue

3

Riven AI executes the tool

Routes to GitHub MCP server, executes with user's OAuth

4

Result returned to model

Model generates final response with issue URL

AI-Driven Tool Calling

Let the AI model automatically discover and use available tools:

bash
curl -X POST http://localhost:8080/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-User-ID: user_123" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [
      {"role": "user", "content": "Create a GitHub issue titled Bug: Login fails for the repo acme/app"}
    ],
    "tools": "auto"
  }'

With tools: "auto", Riven AI automatically includes all available MCP tools the user has access to.

Response with Tool Calls

When the model decides to use a tool, you'll receive:

json
1{
2  "id": "chatcmpl-abc123",
3  "choices": [
4    {
5      "index": 0,
6      "message": {
7        "role": "assistant",
8        "content": null,
9        "tool_calls": [
10          {
11            "id": "call_xyz789",
12            "type": "function",
13            "function": {
14              "name": "github__create_issue",
15              "arguments": "{\"repo\": \"acme/app\", \"title\": \"Bug: Login fails\", \"body\": \"Users are unable to log in...\"}"
16            }
17          }
18        ]
19      },
20      "finish_reason": "tool_calls"
21    }
22  ]
23}

Auto-Execute Mode

Let Riven AI automatically execute tool calls and continue the conversation:

bash
curl -X POST http://localhost:8080/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-User-ID: user_123" \
  -H "X-MCP-Auto-Execute: true" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [
      {"role": "user", "content": "Create a GitHub issue for the login bug"}
    ],
    "tools": "auto"
  }'

# Response (tool already executed, final answer returned)
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "I\'ve created the GitHub issue for you. You can view it here: https://github.com/acme/app/issues/42"
      },
      "finish_reason": "stop"
    }
  ],
  "tool_executions": [
    {
      "tool": "github__create_issue",
      "result": {"issue_url": "https://github.com/acme/app/issues/42", "issue_number": 42}
    }
  ]
}
Auto-execute mode runs tools without user confirmation. Only enable this for trusted users and non-destructive operations.

Direct Tool Invocation

Call MCP tools directly without going through an AI model:

bash
curl -X POST http://localhost:8080/mcp/v1/tools/call \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-User-ID: user_123" \
  -d '{
    "server": "github",
    "tool": "create_issue",
    "arguments": {
      "repo": "acme/app",
      "title": "Bug: Login fails",
      "body": "Users are unable to log in after the latest update."
    }
  }'

# Response
{
  "result": {
    "issue_url": "https://github.com/acme/app/issues/42",
    "issue_number": 42,
    "created_at": "2024-11-15T10:30:00Z"
  }
}

Listing Available Tools

Discover what tools are available for a user:

bash
curl http://localhost:8080/mcp/v1/tools \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-User-ID: user_123"

# Response
{
  "tools": [
    {
      "name": "github__create_issue",
      "description": "Create a new issue in a GitHub repository",
      "server": "github",
      "parameters": {
        "type": "object",
        "properties": {
          "repo": {"type": "string", "description": "Repository in owner/name format"},
          "title": {"type": "string", "description": "Issue title"},
          "body": {"type": "string", "description": "Issue body (markdown)"},
          "labels": {"type": "array", "items": {"type": "string"}}
        },
        "required": ["repo", "title"]
      }
    },
    {
      "name": "slack__send_message",
      "description": "Send a message to a Slack channel",
      "server": "slack",
      "parameters": {
        "type": "object",
        "properties": {
          "channel": {"type": "string"},
          "text": {"type": "string"}
        },
        "required": ["channel", "text"]
      }
    }
  ]
}

Tool Permissions

Control which tools users can access via virtual key configuration:

config.json
1{
2  "virtual_keys": {
3    "mobile-app-key": {
4      "mcp": {
5        "allowed_servers": ["github", "slack"],
6        "allowed_tools": [
7          "github__list_issues",
8          "github__get_issue",
9          "slack__send_message"
10        ],
11        "denied_tools": [
12          "github__delete_repo",
13          "slack__delete_channel"
14        ],
15        "auto_execute": false,
16        "max_tool_calls_per_request": 5
17      }
18    }
19  }
20}

Error Handling

Tool call errors are returned in a structured format:

json
1{
2  "error": {
3    "type": "tool_execution_error",
4    "tool": "github__create_issue",
5    "code": "permission_denied",
6    "message": "User does not have write access to repository acme/private-repo",
7    "details": {
8      "required_permission": "write",
9      "user_permission": "read"
10    }
11  }
12}
