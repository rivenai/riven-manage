MCP Servers

Configure and manage connections to MCP-compatible tool servers.

MCP servers provide tools that AI models can use to interact with external services. Riven AI supports both official MCP servers and custom implementations.

Overview

Riven AI acts as an MCP gateway, connecting to multiple MCP servers and exposing their tools through a unified API. This allows AI models to use tools from GitHub, Slack, databases, and custom services seamlessly.

GitHub

Issues, PRs, repos, code search, and more.

Slack

Send messages, manage channels, search history.

PostgreSQL

Query databases with natural language.

Configuration

Configure MCP servers in your config.json:

config.json
1{
2  "mcp": {
3    "servers": {
4      "github": {
5        "type": "stdio",
6        "command": "npx",
7        "args": ["-y", "@modelcontextprotocol/server-github"],
8        "env": {
9          "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
10        }
11      },
12      "slack": {
13        "type": "stdio",
14        "command": "npx",
15        "args": ["-y", "@modelcontextprotocol/server-slack"],
16        "env": {
17          "SLACK_BOT_TOKEN": "${SLACK_BOT_TOKEN}"
18        }
19      },
20      "postgres": {
21        "type": "stdio",
22        "command": "npx",
23        "args": ["-y", "@modelcontextprotocol/server-postgres"],
24        "env": {
25          "POSTGRES_CONNECTION_STRING": "${DATABASE_URL}"
26        }
27      },
28      "custom-api": {
29        "type": "http",
30        "url": "https://your-mcp-server.com/mcp",
31        "headers": {
32          "Authorization": "Bearer ${CUSTOM_API_KEY}"
33        }
34      }
35    }
36  }
37}

Server Types

stdio Servers

Run MCP servers as local processes communicating via stdin/stdout:

json
1{
2  "type": "stdio",
3  "command": "npx",
4  "args": ["-y", "@modelcontextprotocol/server-github"],
5  "env": {
6    "GITHUB_TOKEN": "${GITHUB_TOKEN}"
7  },
8  "cwd": "/path/to/working/directory",
9  "restart_on_failure": true,
10  "max_restarts": 3
11}

HTTP Servers

Connect to remote MCP servers over HTTP:

json
1{
2  "type": "http",
3  "url": "https://mcp.example.com/api/mcp",
4  "headers": {
5    "Authorization": "Bearer ${API_KEY}",
6    "X-Custom-Header": "value"
7  },
8  "timeout_ms": 30000,
9  "retry": {
10    "max_attempts": 3,
11    "backoff_ms": 1000
12  }
13}

SSE Servers

Connect using Server-Sent Events for long-running operations:

json
1{
2  "type": "sse",
3  "url": "https://mcp.example.com/api/mcp/sse",
4  "headers": {
5    "Authorization": "Bearer ${API_KEY}"
6  }
7}

Managing Servers via API

Add, update, and remove MCP servers at runtime:

bash
# List configured servers
curl http://localhost:8080/api/v1/mcp/servers \
  -H "Authorization: Bearer $ADMIN_KEY"

# Add a new server
curl -X POST http://localhost:8080/api/v1/mcp/servers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_KEY" \
  -d '{
    "name": "jira",
    "type": "http",
    "url": "https://jira-mcp.example.com/mcp",
    "headers": {
      "Authorization": "Bearer jira-token"
    }
  }'

# Check server health
curl http://localhost:8080/api/v1/mcp/servers/github/health \
  -H "Authorization: Bearer $ADMIN_KEY"

# Remove a server
curl -X DELETE http://localhost:8080/api/v1/mcp/servers/jira \
  -H "Authorization: Bearer $ADMIN_KEY"

Health Monitoring

config.json
1{
2  "mcp": {
3    "servers": {
4      "github": {
5        "preset": "github",
6        "health_check": {
7          "enabled": true,
8          "interval_seconds": 60,
9          "timeout_seconds": 10,
10          "unhealthy_threshold": 3
11        },
12        "on_unhealthy": "disable"
13      }
14    },
15    "alerting": {
16      "slack_webhook": "https://hooks.slack.com/...",
17      "alert_on_server_down": true
18    }
19  }
20}

Security Considerations

MCP servers can have significant access to external systems. Always follow the principle of least privilege when configuring server credentials.

Best Practices

  • Use environment variables for all secrets, never hardcode credentials
  • Restrict file system access to specific directories
  • Use read-only database credentials when possible
  • Enable audit logging for all MCP tool calls
  • Review and restrict tool access per virtual key
