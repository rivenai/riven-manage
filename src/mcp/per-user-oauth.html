Riven AI/MCP Gateway/Per-User OAuth

Per-User OAuth

Enable users to connect their own accounts to MCP servers securely.

Per-user OAuth allows each user to authenticate with their own credentials when accessing MCP tools, ensuring proper authorization and audit trails.

Overview

When MCP servers require authentication (like GitHub, Slack, or Google), Riven AI can manage OAuth flows on a per-user basis. This ensures that actions taken through MCP tools are attributed to the correct user and respect their permissions.

Secure Token Storage

Encrypted storage of user OAuth tokens with automatic refresh.

Granular Permissions

Users only grant the scopes they need for each integration.

Audit Logging

Track which user performed which action through MCP.

Configuration

Configure OAuth providers in your config.json:

config.json
1{
2  "mcp": {
3    "oauth": {
4      "providers": {
5        "github": {
6          "client_id": "${GITHUB_CLIENT_ID}",
7          "client_secret": "${GITHUB_CLIENT_SECRET}",
8          "authorization_url": "https://github.com/login/oauth/authorize",
9          "token_url": "https://github.com/login/oauth/access_token",
10          "scopes": ["repo", "read:user"],
11          "redirect_uri": "https://your-riven.com/oauth/callback/github"
12        },
13        "slack": {
14          "client_id": "${SLACK_CLIENT_ID}",
15          "client_secret": "${SLACK_CLIENT_SECRET}",

OAuth Flow

The per-user OAuth flow works as follows:

1

Initiate Connection

User requests to connect an MCP server that requires OAuth

2

Authorization Redirect

Riven AI redirects user to the provider's authorization page

3

User Grants Permission

User reviews and approves the requested scopes

4

Token Exchange

Riven AI exchanges the auth code for access and refresh tokens

5

Secure Storage

Tokens are encrypted and stored, associated with the user

Initiating OAuth

Start the OAuth flow for a user via the API:

bash
# Get authorization URL for a user
curl -X POST http://localhost:8080/api/v1/mcp/oauth/authorize \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "user_id": "user_123",
    "provider": "github",
    "scopes": ["repo", "read:user"],
    "redirect_uri": "https://your-app.com/oauth/complete"
  }'

# Response
{
  "authorization_url": "https://github.com/login/oauth/authorize?client_id=...&scope=repo+read:user&state=...",
  "state": "random_state_token"
}

Handling the Callback

After the user authorizes, handle the callback:

bash
# Exchange authorization code for tokens
curl -X POST http://localhost:8080/api/v1/mcp/oauth/callback \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "provider": "github",
    "code": "authorization_code_from_provider",
    "state": "random_state_token"
  }'

# Response
{
  "success": true,
  "user_id": "user_123",
  "provider": "github",
  "connected_at": "2024-11-15T10:30:00Z",
  "scopes": ["repo", "read:user"]
}

Using Connected Accounts

When making MCP tool calls, include the user ID to use their credentials:

bash
curl -X POST http://localhost:8080/mcp/v1/tools/call \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-User-ID: user_123" \
  -d '{
    "server": "github",
    "tool": "create_issue",
    "arguments": {
      "repo": "owner/repo",
      "title": "Bug report",
      "body": "Description of the issue"
    }
  }'
The X-User-IDheader tells Riven AI which user's OAuth credentials to use for the MCP call.

Managing Connections

List and manage a user's OAuth connections:

bash
# List user's connections
curl http://localhost:8080/api/v1/mcp/oauth/connections?user_id=user_123 \
  -H "Authorization: Bearer $API_KEY"

# Response
{
  "connections": [
    {
      "provider": "github",
      "connected_at": "2024-11-15T10:30:00Z",
      "scopes": ["repo", "read:user"],
      "expires_at": null
    },
    {
      "provider": "slack",
      "connected_at": "2024-11-10T08:00:00Z",
      "scopes": ["channels:read", "chat:write"],
      "expires_at": "2024-12-10T08:00:00Z"
    }
  ]
}

# Revoke a connection
curl -X DELETE http://localhost:8080/api/v1/mcp/oauth/connections/github?user_id=user_123 \
  -H "Authorization: Bearer $API_KEY"

Token Storage

Configure where OAuth tokens are stored:

config.json
1{
2  "mcp": {
3    "oauth": {
4      "token_storage": "database",
5      "database": {
6        "type": "postgresql",
7        "connection_string": "${DATABASE_URL}",
8        "table_prefix": "bifrost_oauth_"
9      },
10      "encryption": {
11        "algorithm": "AES-256-GCM",
12        "key": "${OAUTH_ENCRYPTION_KEY}"
13      },
14      "token_refresh": {
15        "enabled": true,
16        "refresh_before_expiry_minutes": 5
17      }
18    }
19  }
20}
