Custom Providers

Register any OpenAI-compatible API as a named provider. This lets you route to self-hosted models (vLLM, SGLang, Ollama), private deployments, or any third-party endpoint alongside managed providers.

Custom providers work with any service that exposes an OpenAI-compatible API — including vLLM, SGLang, LocalAI, LM Studio, and private cloud deployments.

Add via config

json
{
  "providers": [
    {
      "key": "my-vllm",
      "type": "openai_compatible",
      "base_url": "http://my-vllm-server:8000/v1",
      "api_keys": [{"value": "not-needed"}],
      "models": ["meta-llama/Llama-3.1-8B-Instruct"]
    }
  ]
}

Add via API

bash
curl -X POST http://localhost:8080/api/providers \
  -H "Authorization: Bearer admin-token" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "my-vllm",
    "type": "openai_compatible",
    "base_url": "http://my-vllm-server:8000/v1",
    "api_keys": [{"value": "token"}],
    "models": ["meta-llama/Llama-3.1-8B-Instruct"]
  }'

Route to the custom provider

Use the x-model-provider header to pin a request to a specific provider, or let the gateway route automatically based on model name:

bash
# Pin to the custom provider
curl http://localhost:8080/v1/chat/completions \
  -H "x-model-provider: my-vllm" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3.1-8B-Instruct",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Mix with managed providers

Custom providers participate in all gateway features — fallbacks, load balancing, and semantic caching:

json
{
  "fallback_providers": ["my-vllm", "openai"],
  "routing": {
    "strategy": "cost",
    "providers": [
      {"key": "my-vllm", "weight": 0.8},
      {"key": "openai", "weight": 0.2}
    ]
  }
}

Provider config fields

FieldRequiredDescription
keyyesUnique identifier used in routing rules and x-model-provider header
typeyesMust be openai_compatible for custom providers
base_urlyesBase URL of the OpenAI-compatible API endpoint
api_keysyesArray of API key objects with value field
modelsnoList of model names to register under this provider
timeout_msnoPer-request timeout in milliseconds (default: 30000)
headersnoStatic headers to include with every request to this provider
