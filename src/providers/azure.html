Azure OpenAI

Enterprise-grade OpenAI models through Microsoft Azure.

Azure OpenAI provides enterprise security, compliance, and regional data residency for GPT-4, GPT-4o, and embedding models.

Supported Models

ModelContextFeatures
gpt-4o128KText, Vision, Function Calling
gpt-4o-mini128KText, Vision, Function Calling
gpt-48K / 32KText, Function Calling
text-embedding-3-large8KEmbeddings

Configuration

config.json
1{
2  "providers": {
3    "azure": {
4      "endpoint": "https://your-resource.openai.azure.com",
5      "api_key": "${AZURE_OPENAI_API_KEY}",
6      "api_version": "2024-08-01-preview",
7      "deployments": {
8        "gpt-4o": "your-gpt4o-deployment",
9        "gpt-4o-mini": "your-gpt4o-mini-deployment",
10        "text-embedding-3-large": "your-embedding-deployment"
11      }
12    }
13  }
14}

Using Managed Identity

For Azure deployments, use managed identity instead of API keys:

config.json
1{
2  "providers": {
3    "azure": {
4      "endpoint": "https://your-resource.openai.azure.com",
5      "use_managed_identity": true,
6      "api_version": "2024-08-01-preview"
7    }
8  }
9}

Basic Usage

bash
curl -X POST http://localhost:8080/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "model": "azure/gpt-4o",
    "messages": [
      {"role": "user", "content": "Explain quantum computing"}
    ],
    "max_tokens": 1000
  }'

Multiple Regions

Configure multiple Azure regions for load balancing and redundancy:

config.json
1{
2  "providers": {
3    "azure": {
4      "regions": {
5        "us-east": {
6          "endpoint": "https://us-east-resource.openai.azure.com",
7          "api_key": "${AZURE_US_EAST_KEY}"
8        },
9        "eu-west": {
10          "endpoint": "https://eu-west-resource.openai.azure.com",
11          "api_key": "${AZURE_EU_WEST_KEY}"
12        }
13      },
14      "default_region": "us-east",
15      "failover_regions": ["eu-west"]
16    }
17  }
18}

Content Filtering

Azure OpenAI includes built-in content filtering. Configure the behavior:

config.json
1{
2  "providers": {
3    "azure": {
4      "content_filter": {
5        "hate": "high",
6        "sexual": "high",
7        "violence": "medium",
8        "self_harm": "high"
9      }
10    }
11  }
12}
Content filtering settings are configured at the Azure resource level. Contact Microsoft to modify filter configurations.
