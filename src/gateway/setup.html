Gateway Setup

Deploy the Riven AI HTTP gateway — an OpenAI-compatible AI proxy with a built-in web UI, real-time monitoring, and zero-config provider routing.

Quick Start

Get the gateway running in under a minute with Docker or download the binary directly.

# Pull and run the latest Riven AI gateway
docker run -d \
  --name riven \
  -p 8080:8080 \
  -p 8081:8081 \
  -e OPENAI_API_KEY=sk-... \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  ghcr.io/rivenai/gateway:latest

Configuration

Riven AI can be configured via environment variables, a JSON config file, or the web UI.

Environment Variables

VariableDescriptionDefault
RIVEN_PORTHTTP API port8080
RIVEN_UI_PORTWeb UI port8081
OPENAI_API_KEYOpenAI API key-
ANTHROPIC_API_KEYAnthropic API key-
RIVEN_CONFIGPath to config file./config.json

Config File

Create a config.json file for advanced configuration:

config.json
{
  "providers": {
    "openai": {
      "keys": [
        {
          "name": "primary",
          "value": "env.OPENAI_API_KEY",
          "models": ["*"],
          "weight": 1.0
        }
      ],
      "network_config": {
        "max_retries": 3,
        "retry_backoff_initial": 500,
        "retry_backoff_max": 5000
      }
    },
    "anthropic": {
      "keys": [
        {
          "name": "primary",
          "value": "env.ANTHROPIC_API_KEY",
          "models": ["*"],
          "weight": 1.0
        }
      ]
    }
  },
  "virtual_keys": {
    "my-app": {
      "budget": {
        "max_cost_per_day": 100.0
      },
      "rate_limit": {
        "requests_per_minute": 1000
      }
    }
  }
}

Making Requests

Once running, the gateway exposes an OpenAI-compatible API. Simply change your base URL:

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [
      {
        "role": "user",
        "content": "Hello, how are you?"
      }
    ],
    "max_tokens": 100
  }'
