Riven AI/Features/Guardrails

Guardrails

Content moderation and safety controls for AI applications.

Riven AI provides built-in guardrails to filter harmful content, prevent prompt injection, and enforce content policies across all your AI requests.

Overview

Guardrails run as middleware plugins that inspect and optionally modify requests and responses. They can block requests, redact sensitive information, or add warnings to responses.

Content Filtering

Block harmful, toxic, or inappropriate content from being sent or received.

Prompt Injection Detection

Detect and block prompt injection attacks in user inputs.

PII Redaction

Automatically detect and redact personally identifiable information.

Configuration

Enable guardrails in your config.json:

config.json
1{
2  "plugins": {
3    "guardrails": {
4      "enabled": true,
5      "content_filter": {
6        "enabled": true,
7        "categories": ["hate", "violence", "sexual", "self-harm"],
8        "threshold": "medium",
9        "action": "block"
10      },
11      "prompt_injection": {
12        "enabled": true,
13        "action": "block",
14        "log_attempts": true
15      },
16      "pii_redaction": {
17        "enabled": true,
18        "types": ["email", "phone", "ssn", "credit_card"],
19        "action": "redact"
20      }
21    }
22  }
23}

Content Filtering

Content filtering analyzes both input prompts and model responses for harmful content. You can configure which categories to filter and what action to take.

Categories

CategoryDescription
hateContent that expresses hatred toward groups or individuals
violenceContent depicting or promoting violence
sexualSexually explicit or suggestive content
self-harmContent promoting self-harm or suicide
illegalContent promoting illegal activities

Threshold Levels

LevelDescription
lowOnly block very explicit content (more permissive)
mediumBlock moderately harmful content (balanced)
highBlock any potentially harmful content (most restrictive)

Prompt Injection Detection

Prompt injection attacks attempt to override system instructions or extract sensitive information. Riven API uses multiple detection methods to identify and block these attacks.

config.json
1{
2  "plugins": {
3    "guardrails": {
4      "prompt_injection": {
5        "enabled": true,
6        "detection_methods": ["heuristic", "ml_classifier", "llm_judge"],
7        "action": "block",
8        "log_attempts": true,
9        "alert_webhook": "https://your-domain.com/security-alerts"
10      }
11    }
12  }
13}
Prompt injection detection is not 100% accurate. Always implement defense in depth and never rely solely on input filtering for security.

Detection Methods

MethodDescriptionLatency
heuristicRule-based detection using known patterns<1ms
ml_classifierMachine learning model trained on injection samples~5ms
llm_judgeUse an LLM to evaluate if input is malicious~200ms

PII Redaction

Automatically detect and redact personally identifiable information from requests before they are sent to AI providers.

config.json
1{
2  "plugins": {
3    "guardrails": {
4      "pii_redaction": {
5        "enabled": true,
6        "types": ["email", "phone", "ssn", "credit_card", "address", "name"],
7        "action": "redact",
8        "replacement": "[REDACTED]",
9        "log_redactions": true
10      }
11    }
12  }
13}

Custom Rules

Define custom guardrail rules using CEL (Common Expression Language):

config.json
1{
2  "plugins": {
3    "guardrails": {
4      "custom_rules": [
5        {
6          "name": "block_competitor_mentions",
7          "condition": "request.messages.any(m, m.content.contains('competitor-name'))",
8          "action": "block",
9          "message": "Requests mentioning competitors are not allowed"
10        },
11        {
12          "name": "require_system_prompt",
13          "condition": "!request.messages.exists(m, m.role == 'system')",
14          "action": "block",
15          "message": "A system prompt is required"
16        },
17        {
18          "name": "limit_output_tokens",
19          "condition": "request.max_tokens > 4000",
20          "action": "modify",
21          "modifications": {
22            "max_tokens": 4000
23          }
24        }
25      ]
26    }
27  }
28}

Per-Request Override

Override guardrail settings for specific requests using headers:

bash
curl -X POST http://localhost:8080/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-guardrails-disable: prompt_injection" \
  -H "x-guardrails-threshold: low" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
Per-request overrides can be restricted by virtual key permissions. Only trusted keys should be allowed to disable guardrails.
