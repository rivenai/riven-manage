Drop-in Replacement

Replace existing AI SDK connections by changing just the base URL. Keep your code, gain fallbacks and governance.

Riven AI provides an OpenAI-compatible API, which means you can use it as a drop-in replacement for any SDK that supports custom base URLs. No code changes required — just point your existing SDK to Riven AI and gain automatic failover, load balancing, and governance.

Benefits

Zero Code Changes

Keep your existing application code exactly as is. Just change the base URL.

Instant Resilience

Gain automatic retries, fallbacks, and load balancing without any code changes.

Governance Built-in

Add budgets, rate limits, and access controls through configuration.

Migration Examples

Here is how to migrate your existing SDK usage to Riven AI:

OpenAI Python SDK

from openai import OpenAI

client = OpenAI(
    api_key="sk-..."
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello!"}]
)

OpenAI Node.js SDK

import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-...'
});

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Hello!' }]
});

Anthropic SDK

import anthropic

client = anthropic.Anthropic(
    api_key="sk-ant-..."
)

message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello!"}]
)

Using Environment Variables

For easy switching between direct API access and Riven AI, use environment variables:

python
import os
from openai import OpenAI

# Set in your environment:
# OPENAI_BASE_URL=http://localhost:8080/v1  (for Riven AI)
# OPENAI_BASE_URL=https://api.openai.com/v1 (for direct access)

client = OpenAI(
    base_url=os.getenv("OPENAI_BASE_URL", "https://api.openai.com/v1"),
    api_key=os.getenv("OPENAI_API_KEY")
)
