MCP Tool Filtering

Control exactly which MCP tools are exposed to agents — per virtual key, per team, or globally.

Overview

When agents connect to Riven AI's MCP gateway, they receive a list of available tools. MCP Tool Filtering lets you restrict this list using allowlists and denylists at multiple scopes — so a customer-facing agent only sees safe read-only tools while an internal agent has full access.

Configuration Scopes

ScopeApplies ToPriority
GlobalAll requests through this Riven AI instanceLowest
TeamAll virtual keys belonging to a teamMedium
Virtual KeyA single API keyHighest

Allowlist

Only the listed tools will be visible. All others are hidden.

json
{
  "mcp": {
    "tool_filter": {
      "mode": "allowlist",
      "tools": [
        "read_file",
        "list_directory",
        "search_files"
      ]
    }
  }
}

Denylist

All tools are visible except the ones listed.

json
{
  "mcp": {
    "tool_filter": {
      "mode": "denylist",
      "tools": [
        "delete_file",
        "execute_command",
        "write_file"
      ]
    }
  }
}

Per-Virtual-Key Filtering

Apply granular filters when creating or updating a virtual key via the API:

bash
curl -X POST http://localhost:8080/api/virtual-keys \
  -H "Content-Type: application/json" \
  -d '{
    "name": "customer-agent-key",
    "provider": "openai",
    "mcp_tool_filter": {
      "mode": "allowlist",
      "tools": ["search_products", "get_order_status"]
    }
  }'

Wildcard Matching

Use * to match tool name prefixes:

json
{
  "tool_filter": {
    "mode": "allowlist",
    "tools": ["read_*", "list_*"]
  }
}
