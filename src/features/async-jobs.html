Async Jobs

Submit long-running AI tasks and retrieve results later.

Async jobs allow you to submit AI requests that may take minutes or hours to complete, without blocking your application. Perfect for batch processing, document analysis, and complex reasoning tasks.

Overview

The async jobs API follows a submit-poll-retrieve pattern. Submit a job, receive a job ID, and poll for status or configure a webhook to be notified when complete.

Fire and Forget

Submit jobs and retrieve results when ready.

Webhook Callbacks

Get notified when jobs complete via webhooks.

Priority Queues

Prioritize urgent jobs over background tasks.

Submitting Jobs

Submit an async job using the /async endpoint:

bash
curl -X POST http://localhost:8080/async/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [
      {"role": "system", "content": "You are an expert analyst."},
      {"role": "user", "content": "Analyze this 100-page document and provide a detailed summary..."}
    ],
    "max_tokens": 8000
  }'

# Response
{
  "job_id": "job_abc123xyz",
  "status": "queued",
  "created_at": "2024-11-15T10:30:00Z",
  "estimated_completion": "2024-11-15T10:35:00Z"
}

Checking Job Status

Poll the job status endpoint:

bash
curl http://localhost:8080/async/v1/jobs/job_abc123xyz \
  -H "Authorization: Bearer $API_KEY"

# Response (in progress)
{
  "job_id": "job_abc123xyz",
  "status": "processing",
  "progress": 0.65,
  "created_at": "2024-11-15T10:30:00Z",
  "started_at": "2024-11-15T10:30:05Z"
}

# Response (completed)
{
  "job_id": "job_abc123xyz",
  "status": "completed",
  "created_at": "2024-11-15T10:30:00Z",
  "started_at": "2024-11-15T10:30:05Z",
  "completed_at": "2024-11-15T10:34:23Z",
  "result": {
    "choices": [
      {
        "message": {
          "role": "assistant",
          "content": "Here is my analysis of the document..."
        }
      }
    ],
    "usage": {
      "prompt_tokens": 50000,
      "completion_tokens": 4500,
      "total_tokens": 54500
    }
  }
}

Job Statuses

StatusDescription
queuedJob is waiting to be processed
processingJob is currently being executed
completedJob finished successfully
failedJob failed after all retries
cancelledJob was cancelled by the user

Webhook Callbacks

Instead of polling, configure a webhook to receive job completion notifications:

bash
curl -X POST http://localhost:8080/async/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -H "X-Webhook-URL: https://your-app.com/webhooks/riven-api" \
  -H "X-Webhook-Secret: your-webhook-secret" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [{"role": "user", "content": "..."}]
  }'

Priority Queues

Set job priority using the X-Job-Priority header:

bash
# High priority job (processed first)
curl -X POST http://localhost:8080/async/v1/chat/completions \
  -H "X-Job-Priority: high" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{"model": "openai/gpt-4o", "messages": [...]}'

# Low priority background job
curl -X POST http://localhost:8080/async/v1/chat/completions \
  -H "X-Job-Priority: low" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{"model": "openai/gpt-4o", "messages": [...]}'

Priority Levels

PriorityDescription
criticalProcessed immediately, preempts other jobs
highProcessed before normal priority jobs
normalDefault priority (FIFO)
lowBackground jobs, processed when capacity available

Cancelling Jobs

Cancel a queued or in-progress job:

bash
curl -X DELETE http://localhost:8080/async/v1/jobs/job_abc123xyz \
  -H "Authorization: Bearer $API_KEY"

# Response
{
  "job_id": "job_abc123xyz",
  "status": "cancelled",
  "cancelled_at": "2024-11-15T10:32:00Z"
}

Listing Jobs

List all jobs with filtering:

bash
# List recent jobs
curl "http://localhost:8080/async/v1/jobs?limit=20" \
  -H "Authorization: Bearer $API_KEY"

# Filter by status
curl "http://localhost:8080/async/v1/jobs?status=processing" \
  -H "Authorization: Bearer $API_KEY"

# Filter by date range
curl "http://localhost:8080/async/v1/jobs?after=2024-11-15T00:00:00Z&before=2024-11-16T00:00:00Z" \
  -H "Authorization: Bearer $API_KEY"
