Request Processing Pipeline

Understanding how Riven AI processes requests from ingestion to response delivery.

Pipeline Overview

Every request to Riven AI flows through a well-defined pipeline with multiple stages, each designed for specific processing tasks:

1

Request Ingestion

Parse incoming request, validate format

2

Authentication

Validate API key, resolve virtual key

3

Pre-LLM Hooks

Run plugins (guardrails, caching lookup)

4

Provider Selection

Apply routing rules, select provider/model

5

Request Transformation

Convert to provider-specific format

6

LLM Call

Execute request with retries/fallbacks

7

Response Transformation

Normalize to OpenAI format

8

Post-LLM Hooks

Run plugins (logging, cache store)

9

Response Delivery

Stream or return complete response

Request Lifecycle

internal/gateway/handler.go
func (h *Handler) HandleChatCompletion(ctx context.Context, req *ChatRequest) (*ChatResponse, error) {
    // Stage 1: Authentication
    key, err := h.auth.ValidateKey(ctx, req.APIKey)
    if err != nil {
        return nil, ErrUnauthorized
    }

    // Stage 2: Pre-LLM Hooks
    for _, hook := range h.preHooks {
        if resp, err := hook.Execute(ctx, req); err != nil {
            return nil, err
        } else if resp != nil {
            return resp, nil // Cache hit or blocked
        }
    }

    // Stage 3: Provider Selection
    provider, model := h.router.SelectProvider(ctx, req, key)

    // Stage 4: Execute with retries
    resp, err := h.executor.Execute(ctx, provider, model, req)
    if err != nil {
        return nil, err
    }

    // Stage 5: Post-LLM Hooks
    for _, hook := range h.postHooks {
        hook.Execute(ctx, req, resp)
    }

    return resp, nil
}

Streaming Pipeline

For streaming requests, Riven API uses a different code path optimized for low-latency token delivery:

go
func (h *Handler) HandleStreamingChat(ctx context.Context, req *ChatRequest) (<-chan *Chunk, error) {
    // Same auth and pre-hooks...
    
    // Create streaming channel
    chunks := make(chan *Chunk, 100)
    
    go func() {
        defer close(chunks)
        
        stream, err := h.executor.ExecuteStream(ctx, provider, model, req)
        if err != nil {
            chunks <- &Chunk{Error: err}
            return
        }
        
        for chunk := range stream {
            // Transform to OpenAI SSE format
            chunks <- h.transformer.TransformChunk(chunk)
        }
        
        // Async post-hooks for logging
        go h.runPostHooks(ctx, req, accumulatedResponse)
    }()
    
    return chunks, nil
}
